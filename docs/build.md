# Build and dev

## make targets

| Target       | Effect                                                                                                    |
| ------------ | --------------------------------------------------------------------------------------------------------- |
| `make setup` | `rustup target add wasm32-unknown-unknown` + `cargo install wasm-bindgen-cli` (pinned, currently 0.2.121). |
| `make web`   | Build `gitrust-web` for wasm32, run wasm-bindgen, copy `index.html` into `crates/gitrust-web/dist/`.       |
| `make run`   | `make web`, then `cargo run -p gitrust -- serve --web-dist crates/gitrust-web/dist --addr $(ADDR)`.        |
| `make serve` | Same as `run` but skips the WASM rebuild. Use after editing only server/core code.                         |
| `make check` | `cargo check` on the native default-members set, then `cargo check -p gitrust-web --target wasm32-unknown-unknown`. Both must be green before committing. |
| `make fmt`   | `cargo fmt --all`.                                                                                        |
| `make lint`  | `cargo clippy --all-targets -- -D warnings`.                                                              |
| `make clean` | `cargo clean` + `rm -rf crates/gitrust-web/dist`.                                                         |

`ADDR` defaults to `127.0.0.1:3737`. Override on the command line:
`ADDR=0.0.0.0:8080 make run`.

## Web build pipeline

```
        cargo build --target wasm32-unknown-unknown --release
                          │
                          ▼
       target/wasm32-unknown-unknown/release/gitrust-web.wasm   (~1 MB)
                          │
                          ▼
             wasm-bindgen --target web --no-typescript
                          │
                          ▼
       crates/gitrust-web/dist/{gitrust_web.js, gitrust_web_bg.wasm}
                          │
            cp crates/gitrust-web/index.html → dist/index.html
                          │
                          ▼
                    served by gitrust-server
                  (--web-dist crates/gitrust-web/dist)
```

`index.html` boots `/gitrust_web.js`, which `init()`s the bundle. The
init function is generated by `wasm-bindgen-cli` and **must match**
the version of the `wasm-bindgen` runtime crate pulled in by
`dioxus-web` — `Cargo.lock` is the source of truth, the Makefile
pins `WBG_VERSION` to that.

## Why not dioxus-cli (`dx`)?

`cargo install dioxus-cli` pulls in `git2` and transitively
`libssh2-sys`. Its build script invokes `cc`, and on environments
where `pkg-config` returns Termux/Android headers (we develop in a
PRoot Ubuntu chroot), the C build fails on broken `pthread.h`
declarations. `wasm-bindgen-cli` has no such dependencies and
installs cleanly when `PATH` is stripped of the Termux prefix.

The pipeline `dx` would automate is short enough to invoke directly
(see above), so the indirection is not worth the install cost.

## Adding a new endpoint

1. Add the operation in `gitrust-core` — a plain function taking a
   `&Path` and returning `anyhow::Result<T>` where `T: Serialize`.
2. Add a handler + route in `gitrust-server::router` under
   `/api/...`. Use the `ApiError` wrapper to turn `anyhow::Error`
   into 400 + JSON.
3. (If the UI consumes it) add a `Deserialize` mirror struct and a
   `fetch_*` helper in `gitrust-ui`, plus a render fn and a section
   in `App`.
4. `make check` must be green for both targets.

## Adding a UI section

1. Define a `Deserialize` mirror of the server-side response struct
   in `crates/gitrust-ui/src/lib.rs`. Field names must match.
2. Add a `use_resource(move || async move { fetch_X(&path).await })`
   that reads the `current_repo` signal so it auto-refetches.
3. Render via a `match`-on-`Option<Result<T, String>>` helper that
   returns `Element` for the three states (loading / ok / err).
4. Style with classes in `crates/gitrust-ui/style.css` — it's
   `include_str!`-ed into the bundle.
5. `make web` and hard-reload the browser (Ctrl/Cmd-Shift-R) to bust
   the wasm cache.

## Common pitfalls

- **Page renders blank after a UI change.** Open DevTools Console.
  If `dioxus::launch` panics with "No platform feature enabled", the
  `web` feature dropped off the `dioxus` dep in
  `crates/gitrust-web/Cargo.toml`.
- **`cargo build` doesn't pick up the new wasm bundle.** That's not
  a bug — `cargo build` from the workspace root skips `gitrust-web`
  by design (it's not in `default-members`). Use `make web`.
- **Linker errors mentioning `-lgcc_s` or Termux headers.** `cc` in
  PATH is resolving to Termux clang. Make sure Ubuntu's gcc is
  installed and `/usr/bin/cc` exists.
